## 📚 2. TEMEL KAVRAMLAR

### 🔵 A) Tor Protokolü Durum Makinesi

Tor devresi şu temel durumlardan geçer:

| Durum | Açıklama |
|:------|:---------|
| `IDLE` | Bağlantı yok |
| `CONNECTING` | Guard node'a bağlanıyor |
| `HANDSHAKING` | TLS el sıkışması |
| `CIRCUIT_BUILDING` | Devre inşa ediliyor |
| `CIRCUIT_READY` | Devre hazır |
| `TRANSMITTING` | Veri aktarımı |
| `CLOSING` | Bağlantı kapatılıyor |
| `CLOSED` | Kapalı |

Tor ağı, devre kurulumu ve veri iletiminde belirli durum geçişlerine sahip olup bu geçişlerin modellenmesi güvenlik analizi için kritiktir. [1]

### 🔴 B) Invalid State Transition Nedir?

> Bir FSM'de tanımlı olmayan veya güvenlik politikasını ihlal eden durum geçişleridir.

**Örnekler:**
- `IDLE → TRANSMITTING` (handshake atlanması)
- `CIRCUIT_BUILDING → HANDSHAKING` (geri dönüş)
- `CLOSED → TRANSMITTING` (kapalı devreden veri)

Bu tür geçişler, FSM tabanlı güvenlik açığı modellemesinde kritik bulgular olarak değerlendirilmektedir. [2]

### 🟢 C) Model-Driven Testing (MDT)

Model-Driven Security Testing; güvenlik testi, risk odaklı test ve model tabanlı test yaklaşımlarını birleştiren bir metodolojidur. [3]

## 🔬 4. TEKNİK YAPI

### 📐 FSM Formal Tanımı

FSM şu 5-tuple ile tanımlanır:

$$M = (Q, \Sigma, \delta, q_0, F)$$

Burada:
- $$Q$$ = Durum kümesi (IDLE, CONNECTING, ...)
- $$\Sigma$$ = Giriş alfabesi (events)
- $$\delta: Q \times \Sigma \rightarrow Q$$ = Geçiş fonksiyonu
- $$q_0$$ = Başlangıç durumu
- $$F$$ = Kabul durumları kümesi

**Invalid transition:** $$\delta(q_i, \sigma) = \emptyset$$ veya güvenlik politikasını ihlal eden $$\delta(q_i, \sigma) = q_j$$

### 🛡️ Güvenlik Açığı Kategorileri

Tor ağında tespit edilen başlıca saldırı vektörleri:

| Saldırı Tipi | FSM Karşılığı | Risk |
|:-------------|:--------------|:-----|
| Trapper Attack | Deanonymization | Yüksek |
| Traffic Correlation | TRANSMITTING state manipülasyonu | Yüksek |
| Circuit Hijacking | CIRCUIT_BUILDING bypass | Kritik |
| Replay Attack | HANDSHAKING tekrarı | Orta |

Tor ağı, AS-level ve Node-level saldırganlar tarafından deanonymization saldırılarına karşı savunmasız olabilmektedir. [4]

## 🧪 5. TEST STRATEJİSİ

### Test Seviyeleri:

**1️⃣ Unit Test**
- Her FSM durumunu izole test et
- Valid/invalid geçişleri doğrula

**2️⃣ Integration Test**
- Devre kurulum sürecini uçtan uca test et
- Node zinciri geçişlerini kontrol et

**3️⃣ Security Test**
- Fuzzing ile beklenmedik input'lar gönder
- Boundary state'leri zorla

### Test Case Örneği:

````python
# Invalid Transition Test Case
def test_invalid_transition_idle_to_transmit():
    fsm = TorCircuitFSM()
    fsm.set_state("IDLE")
    
    # Bu geçiş INVALID olmalı
    result = fsm.transition("TRANSMIT_DATA")
    
    assert result == TransitionError.INVALID
    assert fsm.current_state == "IDLE"  # State değişmemeli
    assert fsm.security_log.contains("INVALID_TRANSITION")
````
