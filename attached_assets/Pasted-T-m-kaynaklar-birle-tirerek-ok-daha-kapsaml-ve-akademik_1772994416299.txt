## 📚 1. LİTERATÜR ANALİZİ & TEMEL KAYNAKLAR

### 1.1 Tor Protokolü Güvenlik Araştırmaları

Tor ağında deanonymization saldırıları, özellikle timing correlation ve circuit-level manipülasyon üzerine yoğunlaşmaktadır. Alman makamlarının 2024'te gerçekleştirdiği büyük ölçekli timing saldırısı, Tor'un state-level zafiyetlerini gün yüzüne çıkarmıştır. [1]

Cell-sequence tabanlı yeni deanonymization teknikleri, önceki saldırı modellerinin sınırlamalarını aşarak Tor devre protokolünün state geçişlerini hedef almaktadır. Bu teknikler, FSM modellemesinin neden kritik olduğunu açıkça ortaya koymaktadır. [2]

Kullanıcı tarafı deanonymization olaylarının büyük çoğunluğu kriptografik veya routing hatalarından değil, uygulama katmanındaki state yönetim hatalarından kaynaklanmaktadır. [3]

### 1.2 FSM Tabanlı Güvenlik Testi Literatürü

FSM tabanlı conformance testing metodolojileri, protokol implementasyonlarının doğrulanmasında temel araç olarak kullanılmaktadır. Bu yöntemler karmaşıklık, uygulanabilirlik, hata tespit kapasitesi ve test suite uzunluğu açısından kapsamlı biçimde değerlendirilmiştir. [4]

Model-based security testing (MBST), güvenlik gereksinimlerinin erken doğrulanmasını ve model tabanlı otomatik test üretimini mümkün kılmaktadır. Otomotiv endüstrisindeki uygulamalar, bu yaklaşımın protokol güvenliğine nasıl uyarlanabileceğini göstermektedir. [1]

Protokol implementasyonlarının güvenlik doğrulaması için IOLTS (Input/Output Labeled Transition System) uzantıları geliştirilmiş; bu yaklaşım FSM tabanlı güvenlik testinin teorik temelini oluşturmaktadır. [2]

## 🏗️ 3. FORMAL MODEL & MATEMATİKSEL ÇERÇEVE

### 3.1 Invalid Transition Formal Tanımı

**Tanım 1 (Valid Transition):**
$$\delta_{valid}(q_i, \sigma) = q_j \iff (q_i, \sigma, q_j) \in T_{spec}$$

**Tanım 2 (Invalid Transition):**
$$\delta_{invalid}(q_i, \sigma) = \begin{cases} \emptyset & \text{if } \nexists q_j: (q_i, \sigma, q_j) \in T_{spec} \\ q_j & \text{if } (q_i, \sigma, q_j) \notin T_{sec} \end{cases}$$

**Tanım 3 (Security Policy Violation):**
$$\forall (q_i, \sigma, q_j) \in T_{impl}: (q_i, \sigma, q_j) \in T_{spec} \land \neg violates(q_j, \mathcal{P})$$

Burada $$\mathcal{P}$$ güvenlik politikası kümesini, $$T_{spec}$$ spesifikasyon geçiş kümesini, $$T_{impl}$$ implementasyon geçiş kümesini temsil eder.

### 3.2 Conformance Testing Teorisi

FSM tabanlı conformance testing metodolojisine göre test suite $$T$$ şu koşulu sağlamalıdır: [4]

$$\forall imp \in IMP: imp \text{ passes } T \iff imp \text{ conforms to } spec$$

**Test Coverage Metrikleri:**

$$Coverage_{state} = \frac{|Q_{visited}|}{|Q_{total}|} \times 100$$

$$Coverage_{transition} = \frac{|T_{tested}|}{|T_{total}|} \times 100$$

$$Coverage_{invalid} = \frac{|T_{invalid\_detected}|}{|T_{invalid\_total}|} \times 100$$

### 4.2 Çekirdek FSM Implementasyonu

````python
from enum import Enum, auto
from dataclasses import dataclass, field
from typing import Dict, Set, Optional, List
import logging
from datetime import datetime

# ─── Durum Tanımları ───────────────────────────────────────────
class TorState(Enum):
    IDLE              = auto()
    CONNECTING        = auto()
    TLS_HANDSHAKE     = auto()
    CREATE_SENT       = auto()
    CIRCUIT_BUILDING  = auto()
    CIRCUIT_READY     = auto()
    TRANSMITTING      = auto()
    CLOSING           = auto()
    CLOSED            = auto()
    ERROR             = auto()

class TorEvent(Enum):
    CONNECT           = auto()
    TLS_OK            = auto()
    TLS_FAIL          = auto()
    SEND_CREATE       = auto()
    RECV_CREATED      = auto()
    SEND_EXTEND       = auto()
    RECV_EXTENDED     = auto()
    SEND_RELAY_DATA   = auto()
    RECV_RELAY_DATA   = auto()
    SEND_DESTROY      = auto()
    RECV_DESTROY      = auto()
    CIRCUIT_CLOSED    = auto()
    TIMEOUT           = auto()
    ERROR_EVENT       = auto()

# ─── Güvenlik İhlali Kaydı ─────────────────────────────────────
@dataclass
class SecurityViolation:
    timestamp:      datetime
    circuit_id:     str
    from_state:     TorState
    event:          TorEvent
    attempted_state: Optional[TorState]
    severity:       str  # CRITICAL, HIGH, MEDIUM, LOW
    attack_type:    str
    description:    str

# ─── Ana FSM Sınıfı ────────────────────────────────────────────
class TorCircuitFSM:
    
    # Geçerli geçiş tablosu: {(state, event): next_state}
    VALID_TRANSITIONS: Dict = {
        (TorState.IDLE,             TorEvent.CONNECT):          TorState.CONNECTING,
        (TorState.CONNECTING,       TorEvent.TLS_OK):           TorState.TLS_HANDSHAKE,
        (TorState.CONNECTING,       TorEvent.TLS_FAIL):         TorState.ERROR,
        (TorState.CONNECTING,       TorEvent.TIMEOUT):          TorState.ERROR,
        (TorState.TLS_HANDSHAKE,    TorEvent.SEND_CREATE):      TorState.CREATE_SENT,
        (TorState.CREATE_SENT,      TorEvent.RECV_CREATED):     TorState.CIRCUIT_BUILDING,
        (TorState.CREATE_SENT,      TorEvent.TIMEOUT):          TorState.ERROR,
        (TorState.CIRCUIT_BUILDING, TorEvent.RECV_EXTENDED):    TorState.CIRCUIT_READY,
        (TorState.CIRCUIT_BUILDING, TorEvent.SEND_EXTEND):      TorState.CIRCUIT_BUILDING,
        (TorState.CIRCUIT_READY,    TorEvent.SEND_RELAY_DATA):  TorState.TRANSMITTING,
        (TorState.CIRCUIT_READY,    TorEvent.RECV_RELAY_DATA):  TorState.TRANSMITTING,
        (TorState.TRANSMITTING,     TorEvent.SEND_RELAY_DATA):  TorState.TRANSMITTING,
        (TorState.TRANSMITTING,     TorEvent.RECV_RELAY_DATA):  TorState.TRANSMITTING,
        (TorState.TRANSMITTING,     TorEvent.SEND_DESTROY):     TorState.CLOSING,
        (TorState.TRANSMITTING,     TorEvent.RECV_DESTROY):     TorState.CLOSING,
        (TorState.CLOSING,          TorEvent.CIRCUIT_CLOSED):   TorState.CLOSED,
        (TorState.ERROR,            TorEvent.CIRCUIT_CLOSED):   TorState.CLOSED,
    }

    # Saldırı tipi eşleştirme tablosu
    ATTACK_PATTERNS: Dict = {
        (TorState.IDLE,            TorEvent.SEND_RELAY_DATA):  ("CIRCUIT_BYPASS",   "CRITICAL"),
        (TorState.CLOSED,          TorEvent.SEND_RELAY_DATA):  ("REPLAY_ATTACK",    "CRITICAL"),
        (TorState.CLOSED,          TorEvent.RECV_RELAY_DATA):  ("GHOST_CIRCUIT",    "HIGH"),
        (TorState.TLS_HANDSHAKE,   TorEvent.SEND_RELAY_DATA):  ("HANDSHAKE_SKIP",   "HIGH"),
        (TorState.CIRCUIT_BUILDING,TorEvent.SEND_RELAY_DATA):  ("PREMATURE_DATA",   "HIGH"),
        (TorState.TRANSMITTING,    TorEvent.SEND_CREATE):      ("CIRCUIT_HIJACK",   "CRITICAL"),
        (TorState.CREATE_SENT,     TorEvent.SEND_CREATE):      ("CREATE_FLOOD",     "MEDIUM"),
    }

    def __init__(self, circuit_id: str):
        self.circuit_id     = circuit_id
        self.current_state  = TorState.IDLE
        self.history:  List = []
        self.violations: List[SecurityViolation] = []
        self.logger = logging.getLogger(f"TorFSM.{circuit_id}")

    def transition(self, event: TorEvent) -> dict:
        key = (self.current_state, event)
        
        # ── Geçerli geçiş mi? ──────────────────────────────────
        if key in self.VALID_TRANSITIONS:
            prev_state = self.current_state
            self.current_state = self.VALID_TRANSITIONS[key]
            self.history.append({
                "timestamp": datetime.now(),
                "from":      prev_state,
                "event":     event,
                "to":        self.current_state,
                "valid":     True
            })
            return {"status": "OK", "new_state": self.current_state}

        # ── Invalid geçiş tespiti ───────────────────────────────
        attack_type, severity = self.ATTACK_PATTERNS.get(
            key, ("UNKNOWN_INVALID_TRANSITION", "MEDIUM")
        )

        violation = SecurityViolation(
            timestamp        = datetime.now(),
            circuit_id       = self.circuit_id,
            from_state       = self.current_state,
            event            = event,
            attempted_state  = None,
            severity         = severity,
            attack_type      = attack_type,
            description      = f"Invalid: {self.current_state.name} + {event.name}"
        )
        self.violations.append(violation)
        self.logger.warning(f"[{severity}] {attack_type}: {violation.description}")

        return {
            "status":      "INVALID_TRANSITION",
            "attack_type": attack_type,
            "severity":    severity,
            "state":       self.current_state  # State değişmez!
        }

    def get_security_report(self) -> dict:
        return {
            "circuit_id":        self.circuit_id,
            "total_transitions": len(self.history),
            "violations":        len(self.violations),
            "critical_count":    sum(1 for v in self.violations if v.severity == "CRITICAL"),
            "high_count":        sum(1 for v in self.violations if v.severity == "HIGH"),
            "attack_types":      list(set(v.attack_type for v in self.violations)),
            "violation_details": [vars(v) for v in self.violations]
        }
````

### 4.3 Test Suite Implementasyonu

````python
import pytest

class TestTorFSMSecurity:
    
    def setup_method(self):
        self.fsm = TorCircuitFSM("test-circuit-001")

    # ── Valid Geçiş Testleri ────────────────────────────────────
    def test_valid_full_circuit_lifecycle(self):
        """Tam devre yaşam döngüsü testi"""
        events = [
            TorEvent.CONNECT,
            TorEvent.TLS_OK,
            TorEvent.SEND_CREATE,
            TorEvent.RECV_CREATED,
            TorEvent.RECV_EXTENDED,
            TorEvent.SEND_RELAY_DATA,
            TorEvent.SEND_DESTROY,
            TorEvent.CIRCUIT_CLOSED,
        ]
        for event in events:
            result = self.fsm.transition(event)
            assert result["status"] == "OK"
        
        assert self.fsm.current_state == TorState.CLOSED
        assert len(self.fsm.violations) == 0

    # ── Invalid Geçiş Testleri ──────────────────────────────────
    def test_circuit_bypass_attack(self):
        """IDLE → TRANSMITTING bypass saldırısı"""
        result = self.fsm.transition(TorEvent.SEND_RELAY_DATA)
        assert result["status"] == "INVALID_TRANSITION"
        assert result["attack_type"] == "CIRCUIT_BYPASS"
        assert result["severity"] == "CRITICAL"
        assert self.fsm.current_state == TorState.IDLE  # State korunmalı

    def test_replay_attack_closed_circuit(self):
        """Kapalı devre üzerinden replay saldırısı"""
        # Devreyi kapat
        self._build_and_close_circuit()
        # Kapalı devreden veri göndermeye çalış
        result = self.fsm.transition(TorEvent.SEND_RELAY_DATA)
        assert result["attack_type"] == "REPLAY_ATTACK"
        assert result["severity"] == "CRITICAL"

    def test_circuit_hijack_attempt(self):
        """Aktif devre üzerinde hijack girişimi"""
        self._build_circuit()
        self.fsm.transition(TorEvent.SEND_RELAY_DATA)
        result = self.fsm.transition(TorEvent.SEND_CREATE)
        assert result["attack_type"] == "CIRCUIT_HIJACK"
        assert result["severity"] == "CRITICAL"

    def test_handshake_skip_attack(self):
        """TLS handshake atlanarak veri gönderimi"""
        self.fsm.transition(TorEvent.CONNECT)
        self.fsm.transition(TorEvent.TLS_OK)
        result = self.fsm.transition(TorEvent.SEND_RELAY_DATA)
        assert result["attack_type"] == "HANDSHAKE_SKIP"

    def test_state_preserved_on_invalid(self):
        """Invalid geçişte state korunması"""
        initial_state = self.fsm.current_state
        self.fsm.transition(TorEvent.SEND_RELAY_DATA)  # Invalid
        assert self.fsm.current_state == initial_state

    # ── Yardımcı Metodlar ───────────────────────────────────────
    def _build_circuit(self):
        for e in [TorEvent.CONNECT, TorEvent.TLS_OK,
                  TorEvent.SEND_CREATE, TorEvent.RECV_CREATED,
                  TorEvent.RECV_EXTENDED]:
            self.fsm.transition(e)

    def _build_and_close_circuit(self):
        self._build_circuit()
        for e in [TorEvent.SEND_RELAY_DATA,
                  TorEvent.SEND_DESTROY,
                  TorEvent.CIRCUIT_CLOSED]:
            self.fsm.transition(e)
````

### 5.2 Saldırı Detay Tablosu

| Saldırı | FSM Geçişi | Etki | Tespit Yöntemi |
|:--------|:-----------|:-----|:---------------|
| **Circuit Bypass** | `IDLE→TRANSMIT` | Kimlik doğrulama atlatma | State guard kontrolü |
| **Replay Attack** | `CLOSED→TRANSMIT` | Eski oturum tekrarı | Sequence number kontrolü |
| **Circuit Hijack** | `TRANSMIT→CREATE` | Devre ele geçirme | Transition policy check |
| **Handshake Skip** | `TLS→TRANSMIT` | Şifreleme atlatma | State prerequisite check |
| **Ghost Circuit** | `CLOSED→RECV` | Hayalet devre | Lifetime tracking |
| **Create Flood** | `CREATE→CREATE` | DoS saldırısı | Rate limiting + FSM |
| **Premature Data** | `BUILDING→TRANSMIT` | Veri sızıntısı | Build completion check |

Cell-sequence tabanlı deanonymization teknikleri, bu state geçişlerini hedef alarak Tor'un anonimlik garantilerini zayıflatmaktadır. [2]

Kullanıcı tarafı güvenlik açıklarının büyük çoğunluğu, uygulama katmanındaki hatalı state yönetiminden kaynaklanmakta olup FSM tabanlı koruma mekanizmaları bu açıkları kapatabilmektedir. [3]

## 🎯 10. ÖZGÜN KATKI & ARAŞTIRMA BOŞLUĞU

### Literatürdeki Boşluklar:

> 1. ✅ Tor'a **özel** kapsamlı FSM modeli literatürde eksik
> 2. ✅ Invalid transition → saldırı tipi **otomatik eşleştirmesi** yok
> 3. ✅ **Real-time** tespit aracı geliştirilmemiş
> 4. ✅ FSM tabanlı **deanonymization korelasyonu** kurulmamış

Model-based security testing yaklaşımı, güvenlik gereksinimlerinin erken doğrulanmasını sağlamakta ve bu çalışmada Tor'a özgü biçimde uyarlanmaktadır. [1]

FSM conformance testing metodolojileri, protokol implementasyonlarındaki sapmaları sistematik biçimde ortaya çıkarmakta; bu çalışma söz konusu metodolojileri Tor'un güvenlik analizine uygulamaktadır. [4]
